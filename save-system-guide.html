üåê Why Export/Import Instead of Auto-Sync?

GitHub Pages only serves static files - it can't run a backend server or database. Here are your options:

  • Export/Import - Simple, works offline, you control your data
  • Firebase/Cloud DB - Would require API keys and external services
  • Browser Sync - Only works if you use the same browser account on all devices

We chose export/import because it's simple, secure, and doesn't require any external services!

üîß Troubleshooting

Can't import file: Make sure it's a .json file exported from this game
Lost my save: If you didn't export it, saves only exist in that browser's localStorage
Cleared browser data: localStorage is deleted when you clear browser data - always export first!

üì± Best Practices

  • Store saves in cloud storage (Google Drive, Dropbox, etc.)

  • Export before clearing browser data
